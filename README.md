# Висновики до коду аналізу ефективність жадібного алгоритму та алгоритму динамічного програмування

``main.py``

Ось результати виконання коду (локально):

```
find_coins_greedy(113): {50: 2, 10: 1, 2: 1, 1: 1}
find_min_coins(113): {50: 2, 10: 1, 2: 1, 1: 1} 

Greedy(113) time: 2.6226043701171875e-06 seconds
Greedy(226) time: 2.1457672119140625e-06 seconds
Greedy(113113) time: 0.00034999847412109375 seconds

Dynamic(113) time: 0.00010013580322265625 seconds
Dynamic(226) time: 0.00019884109497070312 seconds
Dynamic(113113) time: 0.11210513114929199 seconds
```

|          | Швидкість | Пам'ять |
|----------|-----------|---------|
| Жадібний | O(n)      | O(1)    |
| ДП       | O(S * n)  | O(S)    |

S - сума (число) для якого виконуємо розрахунок
n - кількість опцій, з яких обираємо

На основі отриманих даних можна зробити такі висновки:

- для поточної задачі алгоритми повертають однаковий результат, але не для кожної задачі жадібний алгорит поверне
  найкраще рішення;
- час виконання алгоритму динамічного програмування значно повільніший за жадібний алгоритм.
